01 · Programming / production

co-primary · ~5–6 h/wk.

What this lane is. Not “learning to code” — that is assumed. It is production discipline: the habits that make work reproducible, so that anyone (including future-you) can re-run it and get the identical result. In research and in model-validation this is an enforced professional norm, and doing it visibly well is itself a hireable signal.

The tools, defined. R 4.6 is the primary language, with the risk packages rugarch, gasmodel, qrmtools, ExtremeRisks. Around the code sits the reproducibility stack: git (records every change to your code), renv (locks the exact version of every R package a project uses so it re-runs identically years later; renv::init() sets this up), Quarto (turns code + writing into a clean rendered report), CI / continuous integration (automated checks that run your code on every change and flag breakage), and Zenodo (archives a release and mints a permanent DOI — a citable, frozen version). C++/Rcpp at reading level only.

Setup items (this season) are quick — an evening each — but they underpin everything after. The value is that they exist early, not the hours spent.
